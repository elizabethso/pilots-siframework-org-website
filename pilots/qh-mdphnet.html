MDPHnet Query Health Pilot

The Massachusetts eHealth Institute’s (MeHI) Massachusetts Department of Public Health Network (MDPHnet) project includes the creation, operation and management of a distributed health data network for public health surveillance. Information regarding MDPHnet is here: http://www.maehi.org/what-we-do/hie/mdphnet. The project is perfectly aligned with all aspects of the Query Health Initiative, including development of the operational framework for the network, governance rules, architecture of the network, network administration, and implementation of a functional distributed network to enable population based querying by the Massachusetts Department of Public Health. Our ongoing work in the areas will provide the important lessons learned that are directly relevant to the Query Health Initiative because MDPHnet is using nearly all of the existing Query Health standards for distributed querying networks.

MDPHnet targets three query types/capabilities: 1) diabetes surveillance, 2) Influenza-like illness surveillance, and 3) ad-hoc menu-driven querying. MDPHnet clinical data partners are the Massachusetts League of Community Health Centers (MLCHC) and Atrius Health. For this pilot we will submit several queries on topics agreed to by ONC, MeHI, and the participating MDPHnet partners.

As part of the Query Health pilot we propose to prototype incorporation of the following additional querying capabilities within MDPHnet:

  • Ad-hoc Querying against the Query Health CEDD: This will entail mapping the MDPHnet data model (ESP) to the CEDD and building a prototype to query against the CEDD to demonstration this capability. This capability could be used by others who adopt the CEDD and the lessons learned will prove valuable.
  • Incorporation of Query Envelop Standards: Query Health is developing standards for the query envelope. The project will build the capability to incorporate additional query envelop meta-data, and incorporate those meta-data standards as they become available.
  • Use of HQMF as the Query Language for Ad-hoc Queries: All ad-hoc menu driven queries will be translated from PopMedNet “native” format into Health Quality Measure Format (HQMF) for transmission within the Query Envelope and then back to SQL for execution against the CEDD data model. Note that Query Health will provide an Implementation Guide for HQMF, to include support for required query capabilities (e.g. stratification), a simplified object model for HQMF Translation as well as a sample SQL Translator to translate the Object Model, which will need to be customized for the MDPHNet schema.
