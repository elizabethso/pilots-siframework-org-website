FDA Mini-Sentinel Query Health Pilot

The FDA Mini-Sentinel project is an ongoing distributed network to support public health surveillance of the safety of medical products. The project includes 17 data partners and currently has over 126 million query-able covered lives. The technical architecture and software used to implement the distributed querying system is fully aligned with the architecture proposed by Query Health. The Mini-Sentinel project includes several types of queries, all of which will be included in the pilot: 1) standardized queries of diagnoses, procedures, and drug use stratified by age group, sex, year, and data source (referred to as summary table queries); 2) modular program requests that allow users to input parameter specifications to address common questions such as identification of new users of medical products, background rates, and treatment patterns, and 3) custom coding of complex algorithms to address questions not satisfied in through the prior querying options.

In this pilot we will use the existing distributed network to issue several queries to each of our 17 data partners. The specific queries will be agreed-upon by FDA and ONC to help demonstrate the capabilities of the system and key elements of distributed querying identified by ONC.

Additional elements of the pilot will be to 1) assess, adopt and implement the Query health meta-data standards for the Query Envelop, 2) prototype full implementation of PopMedNet Version 3.0 which is fully compliant with current Query Health standards for distributed querying, and 3) work with at least one clinical data partner (TBD) that has an existing i2b2 installation to pilot end-to-end querying using the i2b2 HQMF adapter.
